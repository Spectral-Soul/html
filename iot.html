4. Cross-layer planes of WSN

In WSN, strict OSI layering is inefficient. Cross-layer design lets layers share info to save energy and improve performance.

Three Cross-Layer Planes:

  • Power Management Plane: Monitors battery across all layers. Physical layer adjusts TX power, Network layer reduces routing frequency, Application layer slows sensing.
  • Mobility Management Plane: Tracks node movement. Data Link adjusts sync, Network updates routes dynamically.
  • Task Management Plane: Balances sensing tasks. Prevents duplicate sensors from working simultaneously.

Benefits: Energy efficiency, better QoS, reduced latency, extended network lifetime.

5. ISO/OSI vs TCP/IP Layers
📐 Diagram: OSI vs TCP/IP Comparison
OSI Session+Presentation+Application merge into TCP/IP Application; OSI Data Link+Physical merge into Network Access.

Example: Sending Email

OSI: App formats → Presentation encrypts → Session opens → Transport (TCP) → Network (IP routes) → Data Link (Ethernet) → Physical

TCP/IP: Application (SMTP) handles all three top layers → Transport (TCP) → Internet (IP) → Network Access (WiFi/Ethernet)

2. Technologies used in Web of Things (WoT)
  • REST: HTTP methods (GET, POST, PUT, DELETE) for device control
  • JSON: Lightweight data format for device communication
  • WebSockets: Full-duplex real-time communication
  • URI/URL: Unique address for every IoT "thing"
  • HTTP/HTTPS: Standard web protocols (secure)
  • CoAP: Constrained protocol using UDP
  • RDF: Semantic web for device discovery
3. Role of Cloud Computing in Real-Time IoT
  • Massive Storage: Databases and data lakes for TBs of sensor data
  • High-Performance Processing: Run ML/AI models on sensor data
  • Scalability: Auto-scale resources as device count grows
  • Real-Time Analytics: Stream processing (Kafka, Kinesis)
  • Remote Access: Access data from anywhere via internet
  • Device Management: OTA firmware updates, provisioning
  • Security: Encryption, authentication, threat detection
4. Solutions for IoT Network Attacks
  • Lightweight Cryptography: ECC, SIMON/SPECK, PRESENT ciphers
  • Authentication: Digital certificates, 2FA, MAC filtering, OAuth
  • Secure Protocols: DTLS, TLS/SSL, IPsec, MQTT with TLS
  • Intrusion Detection (IDS): Network & host-based anomaly detection
  • Secure Boot & Firmware: Code signing, OTA updates, rollback protection
  • Network Segmentation: VLANs, firewalls, DMZ for IoT devices
  • Physical Security: Tamper-evident enclosures, secure elements
  • Regular Updates: Patch management, vulnerability scanning
  • Data Privacy: Encryption at rest, data minimization, GDPR compliance

CONTINUOUS ASSESSMENT TEST - III

Date: 04.05.2026 | Max Marks: 25 | Duration: 50 mins
1. Challenges in Internet of Vehicles (IoV)
  • High Mobility & Handover: Fast-moving vehicles cause frequent cell tower switches; packet loss risks
  • Ultra-Low Latency: Safety apps need <10ms; 100ms delay at 100 km/h = 2.8 m traveled
  • Security & Privacy: Hijacking risk, location tracking, message spoofing, DoS attacks
  • Scalability: Thousands of vehicles, each sending 1-10 msgs/sec → channel saturation
  • Heterogeneity: DSRC vs C-V2X vs 5G-V2X — different manufacturer protocols
  • Infrastructure: High cost of Road Side Units (RSUs); rural coverage gaps
  • Reliability: Harsh environments, weather effects, EM interference
  • Regulatory: Spectrum allocation, cross-border issues, liability
  • Cost: OBU deployment cost, infrastructure maintenance
2. Role of Blockchain in IoV

Cars (V2V) + RSU (V2I) → Blockchain ledger for secure trust.

Key Roles:

  • Security & Trust: Immutable ledger, tamper-proof vehicle logs

  • Smart Contracts: Auto-payments for tolls, parking, EV charging; auto insurance claims
  • Secure Data Sharing: Verified traffic/road condition data, prevent fake alerts
  • Supply Chain: Track cargo, verify parts provenance
  • Privacy: Selective disclosure, zero-knowledge proofs
